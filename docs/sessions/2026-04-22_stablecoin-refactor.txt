─── KEY DECISIONS (USER-APPROVED) ──────────────────────────────────────────────

  Pricing model
    - Flat USD. No more SOL/SUI denominated prices (was: 0.5 SOL / 12 SUI / 500
      IKA for Pro) — the parity drift was the exact complaint that kicked off
      this refactor.
    - Two stablecoins: USDC and USDT. User picks at checkout (not at plan
      selection). Circle's mainnet USDC + Tether's mainnet USDT on Solana;
      native Circle USDC + Wormhole-bridged USDT on Sui.
    - Two terms: Annual and Lifetime. Monthly was rejected because true
      auto-debit subscriptions aren't feasible without a backend (no pull
      payments on Solana, and we're deploying to Walrus Sites — static only).
    - Prices: Pro $49 / $149, Team $149 / $399. Free stays at $0.

  Ika chain fate
    - Kept in the ChainSelector as "Coming soon" (disabled button with
      "Soon" badge). User asked for this — the dogfood narrative benefits
      from seeing Ika listed.
    - Modal shows a "Coming when Ika mainnet lives" pane instead of payment
      UI if user clicks Ika.

  Wallet persistence
    - Wallet adapters were already autoConnect on both chains — so that was
      already persistent. The *actual* regression was the chain-selector
      useState resetting on route change. That got a localStorage-backed
      hook (lib/useChainPreference.ts).
